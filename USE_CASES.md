# Сценарии использования системы рейтинга сотрудников

## 1. Назначение документа

Документ описывает основные сценарии использования модуля обработки и визуализации рейтинга сотрудников компании.  
Сценарии охватывают действия пользователей и внешних систем при работе с рейтингами, сотрудниками, импортом данных и аналитикой.

---

## 2. Акторы системы

### 2.1 HR-менеджер
Пользователь, ответственный за загрузку оценок, формирование рейтингов и анализ результатов.

### 2.2 Руководитель отдела
Пользователь, который просматривает рейтинги сотрудников своего подразделения, отслеживает динамику и принимает управленческие решения.

### 2.3 Администратор системы
Пользователь, сопровождающий систему: контролирует запуск, миграции, структуру данных и интеграции.

### 2.4 Внешняя система
Сторонняя корпоративная система, которая может передавать рейтинговые данные через API или CSV.

---

## 3. Основные сценарии использования

---

## UC-01. Просмотр списка сотрудников

### Цель
Получить перечень сотрудников с возможностью фильтрации и постраничного просмотра.

### Акторы
- HR-менеджер
- Руководитель отдела
- Администратор

### Предусловия
- система запущена;
- данные о сотрудниках уже существуют в базе.

### Основной поток
1. Пользователь отправляет запрос на получение списка сотрудников.
2. Система получает параметры фильтрации:
   - отдел;
   - активность сотрудника;
   - номер страницы;
   - размер страницы.
3. Система извлекает данные из базы.
4. Система возвращает список сотрудников и метаданные пагинации.

### Постусловия
- пользователь получает актуальный список сотрудников.

### Альтернативные потоки
- если сотрудники отсутствуют, система возвращает пустой список;
- если указан некорректный параметр страницы, API возвращает ошибку валидации.

### Пример API
```http
GET /api/v1/employees/?page=1&size=10
```

## UC-02. Импорт рейтинговых данных из CSV

### Цель
Загрузить в систему пакет оценок сотрудников из CSV-файла за выбранный период.

### Акторы
- HR-менеджер
- Администратор
- Внешняя система

### Предусловия
- система запущена;
- в системе уже существуют сотрудники и метрики;
- пользователь подготовил CSV в корректном формате.

### Основной поток
1. Пользователь выбирает CSV-файл с оценками.
2. Пользователь указывает период, к которому относятся данные.
3. Система принимает файл.
4. Система проверяет формат файла и обязательные поля:
   - `employee_id`
   - `metric_id`
   - `score`
   - `category`
   - `period`
5. Система преобразует строки CSV во внутренние объекты.
6. Система валидирует значения:
   - существование идентификаторов;
   - диапазон баллов;
   - корректность даты;
   - допустимость категории.
7. Валидные записи сохраняются в базу данных.
8. Система возвращает результат импорта:
   - число всех строк;
   - число успешно загруженных строк;
   - число ошибок;
   - список ошибок.

### Постусловия
- корректные записи добавлены в систему;
- пользователь получил отчёт об импорте.

### Альтернативные потоки
- если файл имеет неверное расширение — система возвращает ошибку;
- если файл пустой — система возвращает ошибку;
- если часть строк некорректна — система сохраняет только валидные записи и возвращает ошибки для остальных строк.

### Пример API
```http
POST /api/v1/ratings/import?period=2024-04-01
```
