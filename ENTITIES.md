# Основные сущности системы рейтинга сотрудников

## 1. Назначение документа
Документ описывает ключевые сущности (domain entities) системы, их атрибуты, связи и назначение в рамках модуля расчёта и визуализации рейтинга сотрудников.

---

## 2. Модель предметной области (кратко)

Система решает задачу:
- хранения справочников (отделы, сотрудники, метрики оценки);
- хранения первичных оценок за период (entries);
- расчёта агрегированных результатов (snapshots) для быстрого отображения рейтинга и аналитики;
- отображения динамики рейтинга по периодам.

---

## 3. Перечень сущностей

### 3.1 Department (Отдел)

**Назначение:**  
Справочник подразделений компании. Используется для группировки сотрудников и расчёта ранга внутри отдела.

**Ключевые поля:**
- `id: UUID` — уникальный идентификатор отдела.
- `name: string` — название отдела (например, `Engineering`).
- `code: string` — короткий уникальный код (например, `ENG`).
- `description: string | null` — описание отдела.
- `is_active: bool` — признак активности (для “мягкого” отключения).
- `created_at, updated_at: datetime` — даты создания/обновления (служебные).

**Связи:**
- `Department 1 — N Employee`: один отдел содержит много сотрудников.

---

### 3.2 Employee (Сотрудник)

**Назначение:**  
Справочник сотрудников компании. Сотрудник принадлежит одному отделу и участвует в рейтинге.

**Ключевые поля:**
- `id: UUID` — уникальный идентификатор сотрудника.
- `department_id: UUID` — ссылка на отдел.
- `full_name: string` — ФИО.
- `email: string` — корпоративный email (уникальный).
- `position: string` — должность (например, `Senior Engineer`).
- `hire_date: date` — дата найма.
- `is_active: bool` — активен ли сотрудник в компании.
- `created_at, updated_at: datetime` — служебные даты.

**Связи:**
- `Employee N — 1 Department`: сотрудник принадлежит одному отделу.
- `Employee 1 — N RatingEntry`: у сотрудника много оценок по метрикам.
- `Employee 1 — N RatingSnapshot`: у сотрудника много снимков рейтинга по периодам.

---

### 3.3 RatingCategory (Категория оценки)

**Назначение:**  
Перечисление (enum), позволяющее классифицировать оценки и метрики по типу (качество, командная работа и т.д.).  
Используется для фильтрации/аналитики и унификации метрик.

**Примеры значений:**
- `performance`
- `teamwork`
- `initiative`
- `quality`
- `leadership`

**Связи:**
- Используется в `RatingMetric` и `RatingEntry`.

---

### 3.4 RatingMetric (Метрика оценки)

**Назначение:**  
Справочник метрик, по которым выставляются оценки (например: “Качество кода”, “Командная работа”).  
Метрика имеет вес, участвующий в расчёте итогового балла.

**Ключевые поля:**
- `id: UUID` — уникальный идентификатор метрики.
- `name: string` — название метрики.
- `category: RatingCategory` — категория метрики.
- `weight: decimal` — вес метрики в итоговом расчёте (например, `0.300`).
- `description: string | null` — описание метрики.
- `is_active: bool` — активна ли метрика.
- `created_at, updated_at: datetime` — служебные даты.

**Связи:**
- `RatingMetric 1 — N RatingEntry`: по метрике создаётся много оценок.

---

### 3.5 RatingEntry (Запись оценки)

**Назначение:**  
Первичная оценка сотрудника по конкретной метрике за конкретный период.  
Это “сырые данные”, из которых строится итоговый рейтинг.

**Ключевые поля:**
- `id: UUID` — уникальный идентификатор записи.
- `employee_id: UUID` — ссылка на оцениваемого сотрудника.
- `reviewer_id: UUID | null` — кто выставил оценку (опционально).
- `metric_id: UUID` — ссылка на метрику.
- `period: date` — период оценки (обычно первый день месяца: `YYYY-MM-01`).
- `score: decimal` — балл (обычно диапазон `0..10`).
- `category: RatingCategory` — категория (дублируется для быстрого доступа/аналитики).
- `comment: string | null` — комментарий.
- `created_at, updated_at: datetime` — служебные даты.

**Особенности:**
- может создаваться вручную через API или массово через импорт CSV;
- используется при построении `RatingSnapshot`.

**Связи:**
- `RatingEntry N — 1 Employee`
- `RatingEntry N — 1 RatingMetric`

---

### 3.6 RatingSnapshot (Снимок рейтинга)

**Назначение:**  
Предрассчитанный итог рейтинга по каждому сотруднику за период.  
Хранится отдельно, чтобы не вычислять рейтинг “на лету” при каждом запросе (оптимизация чтения и масштабирования).

**Ключевые поля:**
- `id: UUID` — идентификатор снимка.
- `employee_id: UUID` — ссылка на сотрудника.
- `period: date` — период расчёта.
- `total_score: decimal` — итоговый взвешенный балл за период.
- `rank: int` — место в общем рейтинге (глобальный ранг).
- `department_rank: int` — место в рейтинге внутри отдела.
- `percentile: decimal` — перцентиль (например, топ 10%).
- `delta_score: decimal | null` — изменение итогового балла относительно прошлого периода.
- `delta_rank: int | null` — изменение позиции в рейтинге относительно прошлого периода.

**Особенности:**
- строится командой `POST /ratings/snapshots/build?period=...`;
- кэшируется в Redis для ускорения отдачи.

**Связи:**
- `RatingSnapshot N — 1 Employee`
- (логическая связь с `RatingEntry` через период: снимок агрегирует множество записей оценок).

---

## 4. Схема связей (ER-вид в текстовом виде)

```text
Department (1) ─── (N) Employee
Employee   (1) ─── (N) RatingEntry
RatingMetric (1) ─ (N) RatingEntry
Employee   (1) ─── (N) RatingSnapshot

RatingSnapshot агрегирует RatingEntry по:
- employee_id
- period
- метрикам и их весам
