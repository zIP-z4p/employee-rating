# Инструменты развертывания

## 1. Назначение документа
Документ описывает инструменты и подходы, используемые для развертывания (deployment) приложения локально и на сервере, включая контейнеризацию, конфигурацию окружений, миграции БД и базовые практики эксплуатации.

---

## 2. Контейнеризация и оркестрация

### 2.1 Docker
**Docker** используется для упаковки сервисов в контейнеры и обеспечения воспроизводимого окружения.

**Почему Docker:**
- одинаковый запуск на разных машинах (macOS/Linux/Windows);
- изоляция зависимостей (Python, Node, PostgreSQL, Redis);
- упрощение CI/CD (сборка образов, прогон тестов);
- переносимость на сервер/облако.

В проекте используются отдельные контейнеры для:
- backend (FastAPI);
- frontend (React/Vite → статическая сборка в nginx);
- postgres (PostgreSQL);
- redis (Redis).

---

### 2.2 Docker Compose
**Docker Compose** используется для запуска и управления набором контейнеров как единого стека.

**Зачем Compose:**
- запуск всех сервисов одной командой;
- декларативное описание связей, портов, переменных окружения, volume’ов;
- удобство локальной разработки и тестового окружения.

**Ключевые возможности:**
- `depends_on` для порядка старта сервисов;
- `healthcheck` для проверки готовности Postgres/Redis;
- `volumes` для сохранения данных БД/Redis между перезапусками;
- проброс портов для доступа с хоста.

---

## 3. Управление конфигурацией (Environment Variables)

### 3.1 .env файл
Файл `.env` хранит параметры окружения:
- логин/пароль и имя БД;
- секретный ключ;
- debug-флаг и URL Redis.

Compose автоматически подхватывает `.env`.

**Пример:**
```env
POSTGRES_USER=rating_user
POSTGRES_PASSWORD=rating_pass_local
POSTGRES_DB=rating_db
SECRET_KEY=local-dev-secret-key-change-in-production-min-32-chars
REDIS_URL=redis://redis:6379/0
DEBUG=true
